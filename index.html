Traffic Rule Enforcement System Overview

This system utilizes high-resolution cameras installed at traffic intersections to capture live vehicle data. Detections are processed in real time using YOLOv10 models for vehicle and helmet localization and PaddleOCR for precise Automatic Number Plate Recognition (ANPR).

Features & Verification Pipeline

  • Rider Helmet Detection: Evaluates riders and heads on two-wheelers. Triggers automatic fines for non-compliance instantly.
  • Speed Vector Calculations: Measures coordinate displacement across pixel planes translated to physical meters via homography calibration matrices.
  • Intersection Signal Jumping: Cross-references zebra crossing coordinates with signal phase changes.
  • Central Database Verification: Retrieves real-time Registration Certificates (RC), Insurance expiry, and Fitness Certificates (FC) via REST API.
  • IoT Cloud Dispatcher: Publishes MQTT payloads to AWS/Azure platforms for notifications.

How Speed is Tracked

Speed tracking works via **Ground Plane Homography Matrix Calibration**. A standard 2D camera image represents a distorted perspective projection of the 3D road.

Scale (S) = PhysicalROI / Pixels
d_m = Pixels_displacement * Scale
t_s = Frames_Displaced / FPS
Speed = (d_m / t_s) * 3.6 km/h

By tracking the vehicle's bottom-center reference point (tires contacting the road) with **ByteTrack**, the system computes frame-by-frame coordinate progression, filters noise via **Kalman filters**, and extracts the exact physical velocity.
